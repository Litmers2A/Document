## 원문 (Raw Feedback)

### 1. 잉크 충전 레이아웃 변경 요청

> 현재 잉크/코인 충전 페이지가 **그리드 카드형 레이아웃(2열)**으로 구성되어 있는데,
>
>
> 이를 **세로 카드 리스트형 레이아웃**으로 변경하고 싶어.
>
> 목표는 **첨부한 다른 스크린샷(jpeg)이 아니라,
> 기존 충전소 화면처럼 한 줄에 하나씩 쌓이는 리스트 구조**야.
>
> 변경 요구사항은 아래와 같아:

**1. 전체 구조**

- 카드들을 **1열 세로 리스트**로 배치
- 화면 너비를 꽉 채우는 **가로형 카드**
- 스크롤 시 위에서 아래로 자연스럽게 가격 비교 가능하도록

**2. 카드 레이아웃**

- 좌측:
    - `C 100`, `C 140` 같은 **기본 충전 코인 수량**
    - `+ C 20 보너스` 형태의 보너스는 **보조 텍스트로 함께 노출**
- 우측:
    - 할인 전 가격은 취소선
    - 할인율(%)과 실제 결제 금액을 **강조**
- 모든 카드는 **높이 동일**하게 유지

**3. 정보 위계**

- 사용자가 위에서 아래로 내려가며:
    1. 코인 수량
    2. 보너스 여부
    3. 할인율
    4. 실제 결제 금액

        을 한눈에 비교할 수 있도록 구성

- "인기 / 추천"이 필요한 경우 **리스트 아이템 내부에서 배지로 처리**

**4. UX 의도**

- 여러 상품을 빠르게 비교하고 선택하는 것이 목적
- 카드 간 간격은 너무 크지 않게 유지해 **밀도 있는 리스트 UX**
- 선택된 카드는 border / background 컬러로 명확히 구분

**5. 범위**

- 결제 로직, 데이터 구조는 그대로 유지
- **레이아웃과 컴포넌트 구조만 그리드 → 세로 리스트로 변경**

### 2. 스토리 상세 페이지 레이아웃 변경 요청

현재 스토리 상세보기 페이지에서는 **Proposal(제안 내역)**만 단일 리스트로 노출되고 있는데,

이 페이지에서 **제안 내역과 후원(지원) 내역을 모두 확인할 수 있도록 레이아웃을 확장**하고 싶어.

변경의 목표는:

- 스토리에 대해 **누가 어떤 제안을 했는지**
- **누가 얼마나 후원했는지**

    를 한 페이지에서 명확하게 보여주는 것이야.

**1. 전체 구조 변경**

- `Proposals` 영역을 단일 섹션이 아니라,
    - **제안(Proposals / Backings)**
    - **후원(Supports )**

        두 개의 명확한 섹션으로 분리

- 두 섹션은:
    - 탭(Proposals | Supports) 형태
- 둘 다 내역이 약 100개 정도 있을 수 있다는 점을 고려해서 ui/ux 구성

**2. Proposal 카드 기본 구성**

각 Proposal 카드에는 아래 정보가 포함되어야 해:

**① 제안자 정보**

- 제안한 사람의 프로필 정보 표시
    - 프로필 이미지(avatar)
    - 닉네임
    - "제안자" 라벨 (필요 시)

**② 제안 내용**

- 제안 텍스트 본문

**③ 지지 현황 요약 (Backings Summary)**

- 해당 제안을 지지한:
    - **Backer 수 (예: 4 Backers)**
    - **총 지지 Ink 합계 (예: 3,200 Ink)**
- 이 영역은 Proposal 카드 하단 또는 메타 정보 영역에 요약 형태로 표시

**2-1. Backers 상세 보기 인터랙션**

- Proposal 카드에서 **Backer 수 / 총 Ink 영역을 클릭 가능**하게 처리
- 클릭 시 아래 중 하나의 방식으로 **Backer 상세 목록 노출**
    - 카드 내부 펼침(expand / collapse)
    - 바텀시트 또는 모달
- Backer 상세 목록에는 각 항목마다:
    - backing(지지)한 사람의 프로필 이미지
    - 닉네임
    - 해당 제안에 기여한 Ink 수량
    - backing(지지) 시점

**2-2. Backer 프로필 UX 규칙**

- 제안자 / Backer 모두 **동일한 프로필 컴포넌트 스타일 사용**
- 작은 avatar + 닉네임 중심의 간결한 구성
- 프로필 클릭 시 해당 유저 프로필 페이지로 이동 가능하도록 고려

**3. 후원(Supports) 섹션**

이 섹션에서는 **스토리에 직접 후원한 내역 리스트**를 보여주고 싶어.

각 후원 항목에는:

- 후원한 Ink 수량
- 후원 시 남긴 메시지(있는 경우)
- 후원 시점

**프로필 정보**

- 각 후원 항목에 **후원한 사람의 프로필 정보 표시**
    - 프로필 이미지
    - 닉네임

**4. 프로필 정보 공통 UX**

- 제안자 / 후원자 모두:
    - 동일한 프로필 컴포넌트 사용
    - 작은 avatar + 닉네임 형태

**5. UX 톤 & 의도**

- 스토리를 둘러싼 **참여 흐름(제안 → 후원 → 지지)**이 자연스럽게 드러나야 함
- 정보는 많아지지만 **읽기 부담은 적게**
- 제안&지지과 후원은 성격이 다르므로 **시각적으로 명확히 구분**

**6. 범위**

- 기존 데이터 구조와 기능은 유지
- **레이아웃, 컴포넌트 구성, 정보 노출 방식만 변경**

### 3. 스토리 상세 페이지의 Main Producers / Sponsored by 섹션

스토리 상세 페이지의 **Main Producers / Sponsored by 섹션 구조는 유지**하되,

각 섹션 내부 UI 위계와 표현 방식을 아래 기준에 맞게 변경하고 싶다.

> ⚠️ 전제 조건
>
> - **유저 상세 페이지는 존재하지 않음**
> - 따라서 **모든 유저 카드 / 리스트 아이템은 클릭 불가(non-interactive)**
> - 이 페이지에서는 정보 표시 역할만 수행

**1️⃣ 전체 구조**

- 섹션 구분은 기존과 동일하게 유지
    - **Main Producers**
    - **Sponsored by**
- 변경 범위는 **각 섹션 내부 레이아웃과 정보 위계(UI)**

**2️⃣ Main Producers – 상세 UI 기획**

**🎯 목적**

- "이 스토리를 실질적으로 만든 사람들"을 한눈에 보여주기
- *상위 기여자(Top 3)**는 상징적으로 강조
- 그 외 기여자는 **공정하고 명확한 리스트 정렬**

**2-1. Top 3 Main Producers (가로 카드형)**

**레이아웃**

- 가로 카드 3개
    - 모바일: 1열 가로 스크롤 또는 3열 축소 카드
- 하단 리스트와 명확히 구분되는 시각적 블록

**카드 구성 요소**

- 프로필 이미지 (상대적으로 큼)
- 닉네임
- **역할 라벨**
    - `PROPOSER` → **오직 제안자 1명에게만 표시**
    - 그 외 Top 3 인원에게는 **역할 라벨 표시하지 않음**
- 기여 Ink 수 (가장 강조되는 정보)
- (선택) 순위 배지
    - 🥇 🥈 🥉 또는 `#1`, `#2`, `#3`

**정렬 기준**

- **제안 + 지지 포함 총 기여 Ink 기준 내림차순**
- proposer가 반드시 1등일 필요 없음

**UX 제약**

- 카드 전체는 **클릭 불가**
- hover / press / link 스타일 적용하지 않음
- 정보 강조 목적의 시각 컴포넌트로만 사용

**2-2. 그 외 Main Producers (리스트형)**

**레이아웃**

- Top 3 카드 영역 하단에 세로 리스트
- 한 줄 = 한 명

**리스트 아이템 구성**

- 작은 avatar
- 닉네임
- **기여 Ink 수 (우측 정렬)**
- (선택) 연한 progress bar로 전체 기여 대비 비율 표현

**역할 규칙**

- proposer는 Top 3 영역에서만 라벨 표시
- 리스트 영역에서는 **모든 인원은 암묵적으로 backer**
- `backer` 라벨을 명시적으로 표시하지 않음 (기본 역할)

**정렬 기준**

- 기여 Ink 수 내림차순
- Top 3 인원은 리스트에서 제외
- 4위부터 순차 표시

**UX 제약**

- 리스트 아이템은 클릭 불가
- 링크/네비게이션 요소 없음

**3️⃣ Sponsored by – UI 방향**

**🎯 목적**

- "직접 제작자는 아니지만, 이 스토리를 지지한 존재"를 보조적으로 노출

**UI 원칙**

- Main Producers보다 시각적 우선순위 낮음
- 카드 형태 사용하지 않음
- **compact list 또는 pill 형태**

**구성 요소**

- 이름 또는 로고
- Ink 수량 (secondary 정보)
- 개인 / 브랜드 구분 가능하되 과도한 강조 없음

**UX 제약**

- 클릭 불가
- 상세 이동 없음
- 정보 표시 전용

**4️⃣ 디자인 톤 요약**

- Main Producers: 기여의 상징성, 위계 강조
- Sponsored by: 보조적, 감사 표기 수준
- 인터랙션보다 **정보 구조와 위계 전달**에 집중
