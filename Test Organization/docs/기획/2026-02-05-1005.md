## 원문 (Raw Feedback)

```markdown
# 소셜 로그인 기반 사용자 온보딩 플로우 명세

## 3. 프로필 완성하기 (Onboarding)

### 3.3 온보딩 처리 항목

#### 닉네임
- 사용자 계정 생성 시 시스템이 자동으로 닉네임을 생성한다.
- 닉네임은 다음 형식으로 생성된다.

형용사 + 명사 + 공백 + # + 숫자

예시:
푸른사과 #4921
조용한별 #8122

- 닉네임은 중복을 허용한다.
- 닉네임은 사용자 식별 용도가 아니며, 내부 식별은 user_id를 사용한다.
- UI에서는 닉네임을 그대로 표시하며, 중복 여부는 노출하지 않는다.
- 사용자는 온보딩 이후 프로필 수정 화면에서 닉네임을 변경할 수 있다.

#### 휴대폰 번호 인증
- 모든 사용자는 휴대폰 번호 인증을 완료해야 한다.
- 인증 방식은 SMS 기반 6자리 인증번호(OTP)이다.
- 인증 완료 전까지 서비스 이용은 제한된다.

## 4. 온보딩 완료 조건

- 아래 조건이 모두 충족되면 온보딩 완료 상태로 처리한다.
  - 소셜 로그인 성공
  - 닉네임 자동 생성 완료
  - 휴대폰 번호 인증 완료
```
