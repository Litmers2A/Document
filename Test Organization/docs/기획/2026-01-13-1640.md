## 원문 (Raw Feedback)

```markdown
# Ink 충전 결제 및 결제 완료 화면 요구사항

## 1. 목적

본 문서는 Ink 충전 결제 플로우에서 **결제하기 화면**과 **결제 완료 화면**을 신규로 추가하기 위한 기능·UI·정책 요구사항을 정의한다. 사용자는 다양한 결제 수단을 통해 Ink를 충전하고, 결제 완료 후 충전 결과와 영수증을 확인할 수 있어야 한다.

---

## 2. 화면 구성 개요

### 2.1 화면 목록

* 결제하기 화면
* 결제 완료 화면

### 2.2 진입 경로

* Ink 잔액 부족 시 CTA 버튼 클릭
* Ink 충전 페이지에서 금액 선택 후 `결제하기` 클릭

---

## 3. 결제하기 화면 요구사항

### 3.1 상단 영역

* 화면 타이틀: `결제하기`
* 뒤로가기 버튼 제공

### 3.2 주문 정보 영역

* 충전 상품 정보 표시

  * Ink 수량 (예: `Ink 5,000`)
  * 보너스 Ink (예: `+ Ink 500 보너스`, 선택 노출)
* 가격 정보

  * 기본 금액
  * 할인/보너스 표기 (예: `+10%`)
  * 최종 결제 금액 (KRW)

### 3.3 결제 수단 선택 영역

* 단일 선택(Radio) 구조
* 결제 수단 목록

  1. 카드 결제

     * 설명 문구: `모든 신용카드 및 체크카드`
  2. 계좌이체

     * 설명 문구: `실시간 계좌이체`
  3. 무통장 입금

     * 설명 문구: `가상계좌 입금`
  4. 간편결제

     * 설명 문구: `토스페이`
* 기본 선택값: 카드 결제

### 3.4 약관 동의 영역

* 체크박스 필수 항목
* 문구 예시:

  * `전자금융거래 이용약관 및 결제대행 서비스 이용약관 동의 (필수)`
* 미동의 시 결제 버튼 비활성화

### 3.5 하단 CTA 영역

* 버튼 라벨: `₩5,000 결제하기`
* 버튼 상태

  * 활성: 결제 수단 선택 + 약관 동의 완료
  * 비활성: 위 조건 미충족
* 버튼 클릭 시 결제 PG 연동 플로우 시작

---

## 4. 결제 완료 화면 요구사항

### 4.1 상태 표시 영역

* 성공 아이콘(체크 아이콘)
* 메인 문구: `결제가 완료되었습니다`
* 서브 문구: `Ink가 충전되었습니다. 이제 작가를 지원하고 창작에 참여해보세요!`

### 4.2 결제 내역 카드

* 충전 상품 정보

  * Ink 수량
  * 보너스 Ink
* 결제 정보

  * 결제 수단
  * 결제 일시 (YYYY.MM.DD HH:mm)
  * 거래 번호
* 결제 금액 강조 표시

### 4.3 충전 결과 영역

* 현재 보유 Ink 표시

  * 예: `5,500 Ink`
* 시각적으로 구분된 카드 형태

### 4.4 하단 CTA 영역

* 버튼 1: `영수증 다운로드`

  * 기능: 결제 영수증 PDF 다운로드
* 버튼 2: `확인`

  * 기능: 이전 화면 또는 메인 화면으로 이동

---

## 5. 정책 및 예외 처리

### 5.1 결제 실패

* 결제 실패 시 에러 토스트 또는 모달 노출
* 실패 사유 메시지 노출 (PG 응답 기반)
* 재시도 버튼 제공

### 5.2 중복 결제 방지

* 결제 요청 중 CTA 버튼 disabled 처리
* 새로고침/뒤로가기 시 결제 상태 서버 검증

### 5.3 데이터 처리

* 결제 완료 시

  * Ink 잔액 즉시 반영
  * 결제 로그 및 거래 번호 저장

---

## 6. 디자인 및 UX 가이드

* 최대 가로폭: 1439px 기준 데스크톱 레이아웃
* 카드형 UI 일관성 유지
* 금액/Ink 수량은 Bold 처리로 시각적 위계 강화
* 주요 CTA는 Black Fill 버튼 사용

---

## 7. 추후 확장 고려사항

* 결제 수단 추가 (네이버페이, 카카오페이 등)
* 자동 충전 옵션
* 결제 내역 히스토리 페이지 연동
```
