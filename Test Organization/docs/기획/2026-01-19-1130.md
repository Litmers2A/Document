# Change Analysis: 작가 페이지 정산 정책 반영

> 생성일시: 2026-01-19 11:30
> 상태: applied

## 원문 (Raw Feedback)

### feedback-250119-3.md: 작가 페이지 정산 정책 반영 변경 사항

**1. 제안 관리 리스트 (Proposal Management)**
- 상태 레이블 변경:
  - null: 검토 대기 → 미결정 (작품 공개 시 자동 환불)
  - deal: 수락됨 → 수락됨 (작품 공개 시 정산 예정)
  - drop: 거절됨 → 거절됨 (작품 공개 시 환불 예정)
- 경고 문구 추가: deal/drop 버튼 클릭 시 컨펌 창(Modal) 노출
  - "수락/거절 시 즉시 독자에게 알림이 발송됩니다. 실제 Ink 정산은 작품이 [공개(Published)] 상태로 전환될 때 한꺼번에 처리됩니다."

**2. 작품 공개 프로세스 (Publishing Flow)**
- 최종 확인 요약 (Publishing Summary):
  - "이 작품을 공개하면 다음과 같이 정산됩니다."
  - ✅ 수락(deal)된 제안 (N건): 총 XXX Ink 정산 예정
  - ⚠️ 미결정(null)된 제안 (M건): 총 YYY Ink 환불 처리
  - 최종 체크박스: "작품 공개 후에는 이 결정들을 수정할 수 없습니다. 계속하시겠습니까?"

**3. 작가 지갑 / 정산 내역 (Creator Wallet)**
- Ink 현황 구분:
  - 정산 예정 금액 (Expected): status: deal + work: open
  - 출금 가능 금액 (Available): work: published 완료된 금액
- 정산 내역 상세: "작품 [제목] 공개에 따른 후원금 정산 완료" 사유 표기

**4. 제안 상세 페이지 (Creator View)**
- 정산 시뮬레이션: "현재 이 제안은 [수락] 상태입니다. 작품 공개 시 수수료를 제외한 ZZZ Ink가 정산됩니다."
