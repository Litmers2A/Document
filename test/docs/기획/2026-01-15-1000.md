# Change Analysis: Story·Episode 구조 재정의 및 참여/철회 기능

> 생성일시: 2026-01-15 10:00
> 상태: applied

## 원문 (Raw Feedback)

### 1. 기획 배경 및 목적

본 기획은 기존의 **연재 중심·플랫폼형 구조**를 제거하고, **에피소드 단위의 참여(Proposal / Backing)와 결과(Published)를 명확히 분리**하기 위해 구조를 재정의한다.

또한 사용자가
- 본인이 참여한 제안/후원을 직접 확인하고
- 참여 중인 상태에서는 철회까지 가능하도록

**참여 관리 중심 UX**를 제공하는 것을 목표로 한다.

### 2. 상위 구조 재정의

- Story는 **여러 Episode를 묶는 컨테이너 역할**만 수행
- Story 자체에는 **상태값이 존재하지 않음**
- ❌ 삭제되는 Story 상태: 연재중, 휴재, 완결
- Episode만 상태를 가짐: OPEN, PUBLISHED

### 3. Episode 상세페이지 구조

**OPEN Episode:**
- 에피소드 본문
- 참여 CTA (제안하기, 후원하기)
- Proposal / Backing / Sponsor **통합 리스트** (탭 분리 없음)
- 정렬: 기여 금액(Ink) 내림차순

**철회하기 버튼 노출 조건:**
- Episode.status === OPEN
- participation.userId === 현재 로그인 유저
- participation.type ∈ { Proposal, Backing }
- Sponsor 철회 불가, PUBLISHED 상태 철회 불가

**PUBLISHED Episode:**
- Main Producers 영역: Proposer, Backer, Sponsor 전원 노출
- 정렬: 총 기여 금액 내림차순, 동일 금액 시 참여 시점
- 노출 정보: 닉네임, 기여 타입, 기여 금액
- ❌ 제거: Proposal/Backing CTA, 철회 버튼

### 6. Story 상세페이지

- ❌ "연재중 / 완결" 관련 UI 및 텍스트 전부 제거

### 7. 마이페이지 IA 확장

- **진행 중인 제안 내역** 메뉴 추가
- 리스트 아이템: Story 제목, Episode 제목, 참여 타입, 참여 금액, 참여 일시, Episode 바로가기, 철회하기 버튼

### 8. 작가 콘텐츠 저장 정책

- 임시저장(Draft) 기능 삭제
- OPEN 상태에서 저장 시 즉시 독자 화면에 반영

### 9. 마이페이지 – 크리에이터 전환

- 모든 사용자 기본 역할: 독자
- **「크리에이터 되기」** 버튼 → 프로필 설정 → 즉시 크리에이터 전환 (심사 없음)
- 크리에이터 프로필 설정: 프로필 사진, 닉네임, 작가 소개글

### 10. 글로벌 네비게이션 구조 정책

- 플랫폼 탐색형 ❌ → 콘텐츠 몰입형 ⭕
- ❌ 독자용 사이드바 전면 삭제
- ❌ 하단 네비게이션 바 전면 삭제
- ✅ 크리에이터 스튜디오 내부에서만 사이드바 허용

### 11. Ink 구매 페이지

- 1 Ink = 100원 (고정)
- 구매 옵션: 10, 20, 30, 100 Ink
- ❌ 보너스 / 할인 / 추천 / 이벤트 전부 제거
- 단순 카드형, 동일 시각적 위계
