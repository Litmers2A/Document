## 원문 (Raw Feedback)

### 1. 정산 관리 화면 구조 변경 (서브탭 미사용)

- 현재 정산 관리 화면에 **보유 금액 정보 + 정산 히스토리**가 한 화면에 함께 노출되고 있음

- 정산 내역이 누적될 경우 가독성 저하 및 화면 복잡도 증가 우려

- 서브탭 UI는 사용하지 않고, **페이지 분리 방식**으로 구조를 단순화

**변경 사항:**

- **(1) 정산 관리 메인 화면 → 보유 내역 전용**

유지: 상단 요약 카드, 보유 금액 영역 (정산 예정/출금 가능), 정산 프로세스 안내

- 제거: 정산 내역 리스트 전체

**(2) 정산 내역 → 별도 페이지로 분리**

- 과거 정산 히스토리 전용 화면

- 진입 방식: "정산 내역 전체 보기 →" 링크로 이동 (서브탭 미사용)

### 2. 신고 버튼 UI 개선 (작가 / 독자 공통)

- 신고 기능은 이미 구현되어 있음

- 아이콘만 노출되어 버튼 인지가 어렵고 클릭 영역이 제한적임

- 신고 아이콘 옆에 **"신고" 텍스트를 함께 노출**

- 단순 아이콘이 아닌 **버튼 형태의 클릭 영역**으로 개선

### 3. 어드민 신고 관리 화면 – Mock 데이터 노출

- 어드민 신고 관리 화면은 이미 존재

- 실제 데이터 연동 전, 화면 확인이 어려운 상태

- **Mock 데이터 기반 신고 리스트를 화면에 노출**

- Mock 항목: 신고 일시, 신고 유형, 신고 대상, 신고자, 처리 상태

### 4. 관리자 대시보드 – 회원 수 카드 UX 개선

**변경 배경:**

- 상단 지표 카드 (전체 회원/전체 작품/총 거래액)와

- 하단 관리 카드 (회원 관리/콘텐츠 관리/결제 관리)가

- 기능적으로 동일한 목적(관리 페이지 진입)을 갖고 있어 중복 UI로 인식됨

**변경 방향:**

- 상단 지표 카드에 '관리 진입 버튼'을 포함시킴

- 하단 관리 카드 영역은 제거

**상세 변경:**

- **전체 회원 카드**: 카드 하단/우측에 `회원 관리 →` 버튼 추가, 클릭 시 회원 관리 페이지로 이동

- **전체 작품 카드**: `콘텐츠 관리 →` 버튼 추가, 클릭 시 콘텐츠 관리 페이지로 이동

- **총 거래액 카드**: `결제 관리 →` 버튼 추가, 클릭 시 결제 관리 페이지로 이동

- 버튼은 명시적 CTA 텍스트 노출 권장
