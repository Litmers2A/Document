## 원문 (Raw Feedback)

```
## 1. 문서 정보
- 문서명: 어드민 페이지 UI/UX 변경요구사항
- 대상 시스템: 관리자(Admin) 웹 페이지
- 변경 유형: UI/UX 개선 및 정보 구조(IA) 개선
- 변경 목적:
  - 필터 사용성 개선
  - 배너 관리 기능 확장
  - 콘텐츠(스토리/에피소드) 관리 구조 명확화

## 2. 변경 배경 및 문제 정의

### 2.1 필터 UI 관련 문제
- 동일 화면 내 필터가 2개 이상 존재할 경우:
  - 필터 UI 형태가 제각각이거나
  - 화면 상단에 필터가 과도하게 노출되어 복잡도가 높아짐
- 필터 간 우선순위 및 관계가 명확하지 않음

### 2.2 배너 관리 기능 한계
- 현재 배너 관리에서 이미지 첨부가 불가능하거나
- 이미지가 배너 미리보기/배경에 반영되지 않아 실제 노출 형태를 예측하기 어려움

### 2.3 콘텐츠 관리 구조 문제
- 스토리(상위 개념)와 에피소드(하위 개념)가 하나의 리스트에 혼재됨
- 관리자가 스토리 단위 관리, 특정 스토리에 속한 에피소드 확인을 직관적으로 수행하기 어려움

## 3. 변경 요구사항 상세

### [CR-01] 필터 UI 통합 (드롭다운 방식)
- 필터가 2개 이상 존재하는 모든 어드민 페이지에서 필터 UI를 단일 드롭다운(Dropdown) 방식으로 통일
- 드롭다운 버튼 예시: `필터` / `상세 필터` / `검색 조건`
- 드롭다운 내부에 각 필터 항목을 세로 리스트 형태로 배치
- 드롭다운 내부에서 조건 선택, `적용` 버튼 클릭 시 리스트 반영, `초기화` 버튼 제공

### [CR-02] 배너 관리 – 이미지 첨부 및 배경 반영
- 배너 관리 페이지에서 이미지 파일 첨부 기능 추가
- 첨부된 이미지는 배너 미리보기 및 실제 배너 배경으로 적용
- 업로드 방식: 파일 선택 또는 드래그 앤 드롭
- 허용 포맷: JPG, PNG, WEBP (확장 가능)
- 업로드 즉시 배너 미리보기 영역에 반영, 텍스트/버튼이 이미지 위에 오버레이 형태로 표시

### [CR-03] 콘텐츠 관리 구조 개편 (스토리 → 에피소드 분리)
- 콘텐츠 관리 페이지에서 스토리 리스트 페이지와 에피소드 리스트 페이지를 명확히 분리
- 스토리 리스트: 스토리 썸네일/제목/상태/에피소드 개수/생성일, 스토리 클릭 시 해당 스토리의 에피소드 리스트로 이동
- 에피소드 리스트: 선택된 스토리 정보 표시, 에피소드 제목/회차/상태/생성일, 에피소드 상세/수정/추가 액션
```
