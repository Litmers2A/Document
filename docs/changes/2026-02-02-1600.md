# 4️⃣ 이미지 크레딧 자동 부착 — 유저 비노출 방식

## 요구 해석

> 유저 화면에는 크레딧 UI가 안 보이지만코드/이미지 자체에는 크레딧이 붙어 있길 원함

## 가능 방식

### 방식 A — 서버 렌더링 시 워터마크 합성

업로드 시:

```
원본 이미지 + 크레딧 텍스트 → 합성 → 저장
```
결과:

- 실제 이미지 파일에 크레딧 포함

- 프론트 UI에는 별도 표시 없음

- 이미지 다운로드 시에도 크레딧 유지

## 워터마크 위치

- 하단 우측

- 투명도 30~40%

- 자동 적용

## 워터마크 구성

```
작가명 + 플랫폼URL
```
## 작가 설정

옵션:

```
[ ] 크레딧 워터마크 적용
```

## 추가 명세 (AI 작업용)

### 1. 참여 내역 (Participation Data)

#### 참여 유형 정의

- 제안: PROPOSAL

- 지지: PROPOSAL_BACKING → BACKER

- 후원: DIRECT_BACKING → SPONSOR

#### 후원 내역(Sponsorship Data) 범위

- 후원 내역에는 **DIRECT_BACKING만 포함**

- PROPOSAL_BACKING은 지지(BACKER)로 분류

- SPONSOR 뱃지는 DIRECT_BACKING을 한 사용자에게만 표시

### 2. 참여 유형 뱃지 로직 (Main Producers 포함)

- 한 사용자는 동시에 여러 참여 유형을 가질 수 있음

PROPOSER

- BACKER

- SPONSOR

사용자 이름 옆에 **복수 뱃지 표시 허용**

기존 `isProposer` 같은 사용자 단위 플래그는 사용하지 않음

참여 유형은 **proposal / backing 데이터 기반으로 판단**

데이터 모델 수정 필요

### 3. Top3 Proposal 블록

#### 표시 기준

- 금액 기준 내림차순 정렬

- 상위 3개 proposal 표시

#### 범위

- 기준 범위는 **작품 단위(story 기준)**

#### 클릭 동작

- Proposal List Modal 오픈

- 표시 내용:

제안 목록

- 제안자

- 누적 금액

- 상태

지지/후원 버튼은 노출하지 않음

### 4. 시리즈 기여자 표시

#### 페이지 위치

- 시리즈 상세 페이지: `/stories/[storyId]`

#### 구분 기준

##### 진행중 기여자

- proposal 또는 backing 존재

- status ≠ DEAL

##### 완료 기여자

- status = DEAL

- 출판 콘텐츠에 반영된 경우

##### 제외 대상

- status = DROP 인 proposal/backing은 표시하지 않음

#### 정렬

- 각 그룹 내 금액 기준 내림차순

### 5. 최근 본 작품 (Recently Viewed Stories)

#### 저장 방식

- **백엔드 DB 기준 (계정 단위 저장)**

#### 노출 규칙

- 메인 홈에 최대 5개 표시

- 최근 열람 순 내림차순

- 5개 초과 시 더보기 페이지 이동

#### 저장 트리거

- 작품 상세 페이지 진입 시 기록 생성 또는 갱신

- 에피소드 열람 시:

신규 추가 ❌

- 기존 기록의 `lastViewedAt` timestamp 갱신

### 6. 이미지 워터마크 (크레딧 자동 부착)

- 서버에서 이미지 생성/저장 시 워터마크 합성

- 워터마크 구성:

### 7. 닉네임 정책
