# Change Analysis: 회원가입 플로우 단계별 분리

> 생성일시: 2026-01-27 09:35
> 상태: pending

## 원문 (Raw Feedback)

### 1. 변경 배경

기존 회원가입 화면에서는
이메일 인증, 비밀번호 설정, 휴대폰 인증, 약관 동의가 **단일 화면에 동시에 노출**되어 있어 다음과 같은 문제가 발생하고 있습니다.

* 입력 항목 과다로 인한 초기 이탈 가능성 증가
* 인증 실패 시 전체 입력 정보 재입력 필요
* 각 인증 단계의 목적(계정 식별 / 신뢰도 검증)이 사용자에게 명확히 전달되지 않음

이에 따라, 회원가입 과정을 **단계별 화면으로 분리**하여
사용자 경험을 개선하고, 인증 단계별 목적을 명확히 하고자 합니다.

### 2. 변경 목적

* 회원가입 초기 진입 장벽 완화
* 인증 실패 시 재시도 비용 최소화
* 이메일 / 휴대폰 인증의 역할을 분리하여 UX 명확화
* 단계별 상태 관리가 가능한 가입 플로우 구조 확보

### 3. 변경 전 / 변경 후 비교

#### 변경 전
* 이메일 인증
* 비밀번호 입력
* 휴대폰 인증
* 약관 동의
→ **단일 화면에서 동시 진행**

#### 변경 후
* 단계별 화면 분리 방식으로 진행
```
이메일 인증
→ 비밀번호 설정
→ 휴대폰 인증
→ 약관 동의
→ 회원가입 완료
```

### 4. 변경 상세 내용

#### 4-1. 이메일 인증 화면 분리
* 이메일 입력 전용 화면 구성
* 인증 메일 발송 및 인증 코드 입력을 단독 단계로 분리
* 이메일 인증 완료 후에만 다음 단계로 이동 가능

**의도**
* 이메일을 계정의 유일한 식별자로 먼저 확정
* 봇 가입 및 중복 가입 방지

#### 4-2. 비밀번호 설정 단계 분리
* 이메일 인증 완료 후 비밀번호 설정 화면 노출
* 비밀번호 / 비밀번호 확인 입력
* 해당 단계에서 계정 생성 처리

**의도**
* 인증되지 않은 사용자 계정 생성 방지
* 로그인 가능 상태를 명확히 정의

#### 4-3. 휴대폰 인증 화면 분리
* 휴대폰 번호 입력 및 인증번호 확인을 별도 화면에서 진행
* 서비스 정책에 따라 필수 또는 선택 처리 가능
* 선택일 경우 "나중에 하기" 옵션 제공 가능

**의도**
* 휴대폰 인증을 신뢰도 확보 수단으로 분리
* 결제/후원/정산 등 거래 행위 이전에 인증 유도 가능

#### 4-4. 약관 동의 화면 분리
* 필수 약관과 선택 약관 구분
* 최종 가입 완료 전 마지막 단계로 배치

**의도**
* 법적 동의 절차의 명확화
* 사용자 인지 부담 최소화

### 5. 기대 효과
* 회원가입 이탈률 감소
* 인증 단계별 실패 원인 명확화
* UX 흐름 단순화 및 가독성 향상
* 단계별 가입 상태 관리 용이
  (예: 이메일 인증 완료 / 휴대폰 인증 미완료 계정)

### 6. 참고 사항
* 단계별 Progress Indicator 적용 권장
  (예: 1/4 이메일 인증, 2/4 계정 설정 등)
* 휴대폰 인증 필수 여부는 서비스 정책에 따라 조정 가능
* 각 단계는 독립적인 API 및 상태값 관리 구조를 전제로 설계 필요

## 영향 범위 분석 (Impact Analysis)

### 화면 (Routes)

| Route | 변경 내용 | 신규 여부 | 상태 |
|-------|----------|---------|------|
| /signup | 스타일 변경: 단일 폼 → Step 컨테이너 레이아웃, 진행 상황 표시 | No | Modified |
| /signup/email | 이메일 입력 및 인증 코드 입력 단계 | Yes | Added |
| /signup/password | 비밀번호 설정 및 확인 단계 | Yes | Added |
| /signup/phone | 휴대폰 번호 입력 및 인증번호 확인 단계 | Yes | Added |
| /signup/terms | 약관 동의 단계 | Yes | Added |
| /signup/complete | 기존 완료 페이지 (유지) | No | No Change |

| Component | 변경 내용 | 신규 여부 | 책임 |
|-----------|----------|---------|------|
| SignupForm | 다중 Step 핸들링 로직으로 재구성, Step 네비게이션 로직 | No | Modified |
| SignupLayout | Progress Indicator 통합, Step 컨테이너 CSS | Yes | Added |
| ProgressIndicator | 1/5, 2/5 등 현재 단계 표시, 완료 상태 시각화 | Yes | Added |
| EmailVerificationStep | 이메일 입력 폼, 인증 코드 입력, 검증 로직 | Yes | Added |
| PasswordSetupStep | 비밀번호 입력, 비밀번호 확인, 규칙 검증 로직 | Yes | Added |
| PhoneVerificationStep | 휴대폰 번호 입력, 인증번호 입력, 선택/필수 토글 | Yes | Added |
| TermsStep | 필수/선택 약관 체크박스, 동의 검증 | Yes | Added |

### 상태/Interaction

| Hook/Store | 변경 내용 | 타입 |
|------------|----------|------|
| useSignupFlow | 새 훅: 현재 단계 상태, 각 단계별 완료 여부, 단계 네비게이션 로직, 임시 저장 | New |
| useEmailVerification | 새 훅: 이메일 인증 로직 (이메일 유효성, 코드 발송, 코드 검증) | New |

| usePhoneVerification | 새 훅: 휴대폰 인증 로직 (번호 유효성, SMS 발송, 코드 검증) | New |

### 스타일/UX

| 항목 | 변경 내용 | 근거 |
|------|----------|------|
| Progress Indicator | 시각적 단계 표시 (1/4, 2/4 등), 완료 표시 | "단계별 Progress Indicator 적용 권장" |
| Form Layout | 한 화면에 1-2개 입력 필드만 노출 | "입력 항목 과다로 인한 초기 이탈 가능성 증가" |
